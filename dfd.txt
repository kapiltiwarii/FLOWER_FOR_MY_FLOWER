Explanation of Level 1 DFD Processes
1. User Authentication:

---> Input: User credentials (username/password).
---> Processes: Verify user credentials, create JWT tokens.
---> Outputs: Authentication token, user profile data.
---> Data Store: User Database.

2. Study Group Management:

---> Input: User requests to create, join, or leave study groups.
---> Processes: Create study group, manage group membership, schedule sessions.
---> Outputs: Updated group information, session schedules.
---> Data Store: Study Group Database.

3. Messaging:

---> Input: Messages from users within study groups.
---> Processes: Store messages, retrieve chat history.
---> Outputs: Real-time message updates.
---> Data Store: Message Database.
---> External Service: Messaging API.

4.Video Conferencing:

---> Input: Requests to start/join video calls.
---> Processes: Establish video connection, manage video sessions.
---> Outputs: Video call links, session information.
---> External Service: Video API.

Explanation of Level 2 DFD Processes
1. Create Group:

---> Input: Group name, description, creator user ID.
---> Processes: Validate data, store new group in the database.
---> Outputs: Confirmation of group creation, group ID.
---> Data Store: Study Group Database.

2. Join Group:

---> Input: User ID, group ID.
---> Processes: Validate group existence, add user to group.
---> Outputs: Confirmation of joining, updated group membership.
---> Data Store: Study Group Database.

3. Schedule Session:

---> Input: Group ID, session details (time, date).
---> Processes: Validate group and time slot, store session details.
---> Outputs: Confirmation of session scheduling.
---> Data Store: Study Group Database.

Summary:
A Data Flow Diagram (DFD) helps in visualizing the flow of data within a system. For the virtual study group platform:

Level 0 DFD provides a high-level view.
Level 1 DFD details key processes such as user authentication, study group management, messaging, and video conferencing.
Level 2 DFD can further break down specific processes like study group management into more granular sub-processes.
This structured approach aids in understanding the interactions within the system and identifying areas for detailed development and optimization.
